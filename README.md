# EduPath — موقع الاستشارات التعليمية للدراسة في ماليزيا

موقع ثابت (Static Site) متعدد الصفحات، بياناته في `localStorage` مع طبقة مزامنة سحابية عبر Supabase — كل الأجهزة تشوف نفس المحتوى الذي يحرره الأدمن.

## الصفحات

| الملف | الوظيفة |
|---|---|
| `index.html` | الرئيسية: خدمات، جامعات، حاسبة تكاليف، أسئلة شائعة، استشارة، بوت محادثة |
| `universities.html` / `university.html` | دليل الجامعات + صفحة تفاصيل الجامعة |
| `courses.html` | بحث في التخصصات والبرامج |
| `institutes.html` | معاهد اللغة |
| `scholarships.html` | المنح الدراسية |
| `blog.html` | المدونة (يدعم رابط مباشر للمقال: `blog.html#post-<id>`) |
| `apply.html` | بوابة الطالب: تسجيل، تقديم طلب، رفع مستندات، متابعة الحالة |
| `admin.html` | لوحة التحكم (كلمة المرور الافتراضية: `edupath2026` — **غيّرها من الإعدادات**) |

## الملفات المشتركة

- `db.js` — طبقة المزامنة مع Supabase (الإصدار 2):
  - عند فتح أي صفحة: يسحب بيانات السحابة → `localStorage` → يطلق حدث `db-ready`.
  - عند أي حفظ لمفتاح مُزامَن: يرفعه للسحابة تلقائياً.
  - `dbSubmit(key, item)`: إضافة عنصر (استشارة/طالب/طلب) مع **دمج آمن** بنسخة السحابة حتى لا تضيع بيانات مرسلة من أجهزة أخرى في نفس الوقت.
  - بيانات الطلاب والطلبات والاستشارات والوسائط تُسحب **فقط** في `admin.html` و`apply.html` (خصوصية + سرعة).
- `shared.js` — القائمة العلوية والفوتر الموحدان لكل الصفحات الداخلية.
- `data.js` — بيانات الجامعات الأساسية (36 جامعة، QS 2026).
- `i18n.js` + ترجمات داخل الصفحات — 5 لغات (EN/AR/JA/KO/RU).
- `app.js` — منطق الصفحة الرئيسية كاملاً.

## ما الذي أصلحته نسخة v2؟ (أخطاء النظام السابق)

1. **مفاتيح مزامنة ناقصة** — تعديلات الأدمن على: إخفاء الجامعات، الأسئلة الشائعة، الخدمات، خطوات التقديم، آراء الطلاب، والتخصصات كانت تُحفظ محلياً فقط ولا تصل للزوار أبداً. أُضيفت كلها لقائمة المزامنة.
2. **الصفحة الرئيسية لا تنتظر السحابة** — كانت تعرض المحتوى الافتراضي حتى لو عدّل الأدمن كل شيء. الآن تعيد العرض تلقائياً عند وصول البيانات (`db-ready`).
3. **فقدان بيانات عند الإرسال المتزامن** — أي استشارة/تسجيل/طلب كان يرفع المصفوفة كاملة فيمسح ما أرسلته أجهزة أخرى. الآن `dbSubmit` يدمج بالـ id قبل الرفع.
4. **خصوصية**: بيانات الطلاب (إيميلات/هواتف/مستندات) كانت تُنزَّل على جهاز كل زائر في كل الصفحات. الآن تُسحب فقط حيث تلزم.
5. **حرف سيريلي مخفي** في أسماء جامعة UniSZA داخل بوت المحادثة كان يعطّل التعرف عليها.
6. **رقم واتساب مثبّت بالكود** في نموذج الاستشارة — الآن يقرأ الرقم من إعدادات لوحة التحكم.
7. **SEO**: أُضيف وصف، أيقونة موقع، بطاقات مشاركة (Open Graph)، و`theme-color` لكل الصفحات + `noindex` للوحة التحكم.
8. مؤشر مزامنة رفيع أعلى الصفحة أثناء تحميل بيانات السحابة.

## ملاحظات أمنية مهمة

- مفتاح Supabase الموجود في `db.js` هو مفتاح عام (publishable) — هذا طبيعي، لكن جدول `store` **قابل للكتابة من أي زائر** لأن الموقع بدون خادم. أي شخص تقني يفتح أدوات المطور يقدر يعدّل محتوى الموقع أو يقرأ بيانات الطلاب.
- كلمة مرور لوحة التحكم حماية شكلية فقط (تتحقق في المتصفح) وهي خاصة بكل جهاز.
- **التوصية للمرحلة القادمة**: تفعيل Supabase Auth + قواعد RLS بحيث: القراءة العامة لمفاتيح المحتوى فقط، والكتابة على المحتوى للأدمن المسجّل فقط، وبيانات الطلاب لا تُقرأ إلا بحساب. أقدر أجهزها لك متى ما حبيت.
